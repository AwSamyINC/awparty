# Дизайн: анимации смерти врагов

Дата: 2026-06-23

## Проблема

Обычные враги при смерти **пропадают резко**: партикл-взрыв и лут уже порождаются,
но само тело врага исчезает тем же кадром — спрайт удаляется сразу после обработки
смерти.

Текущий поток (на кадр):

1. [src/scene_combat.js](../../../src/scene_combat.js) ~97 `handleEnemyDeaths()` —
   для каждого врага с `hp <= 0`: партиклы (15 у обычного, до 300 у боссов), лут,
   гемы, души, shake.
2. [src/scene.js](../../../src/scene.js) ~673
   `_filterDestroy(this.enemies, e => e.hp <= 0)` — **уничтожает спрайт тем же кадром**.

Итог: взрыв есть, но тело «телепортируется» в небытие.

## Цель

Обычные враги при смерти проигрывают короткую анимацию оседания
(«падение/распад вниз»), а не исчезают мгновенно. **Боссов не трогаем** — у них
остаётся резкое исчезновение под собственный взрыв + shake + душу.

Сбор лута/партиклов/душ **не меняем** — он уже работает в `handleEnemyDeaths`.

## Решение: отдельная сущность-«труп» `DeathFx`

Когда обычный враг умирает, снимаем «слепок» его вида из живого `e.sprite` и
порождаем лёгкую отдельную сущность `DeathFx`. Сам враг удаляется как сейчас
(`_filterDestroy` тем же кадром). `DeathFx` держит **собственный** спрайт, проигрывает
анимацию оседания и сам уходит в пул по завершении.

### Почему отдельная сущность, а не флаг `dying` на враге

- **Нулевые правки игровой логики.** Мёртвый враг, оставшийся в `this.enemies`,
  пришлось бы исключать из спатиал-грида, сепарации, ИИ, наведения, попаданий пуль —
  в каждом месте. Инвазивно и багоопасно.
- **Это существующий паттерн проекта.** `Particle`, `DamageText`, `Vinyl` — отдельные
  визуальные сущности в своих массивах: `reinit/update(dt)/release/destroy` + пул.
  `DeathFx` встаёт туда же 1:1.
- Партикл-взрыв и лут уже порождаются в `handleEnemyDeaths` — добавляем лишь
  порождение трупа рядом.

### Порядок исполнения (почему слепок корректен)

`spawnDeathFx(e)` вызывается **внутри** `handleEnemyDeaths`, которое выполняется
**до** `_filterDestroy` (scene.js ~673). Значит `e.sprite` ещё жив — слепок снимется
корректно. Труп создаёт собственный спрайт, поэтому последующее уничтожение врага его
не затрагивает.

## Класс `DeathFx` (в [src/entities.js](../../../src/entities.js))

По образцу `Particle`/`DamageText`: спрайт через `scene.addWorld(...)`, методы
`reinit()` / `update(dt)` / `release()` / `destroy()`, пулится.

**Слепок при порождении** (читается из живого `e.sprite`):
текстура + кадр, `scaleX/scaleY`, `displayHeight`, угол, позиция `x, y`.

Спрайт трупа ставится с **origin (0.5, 1.0)** в точке «ног» врага
(`y + displayHeight/2`) — тогда вертикальное сжатие читается как оседание **к земле**,
а не схлопывание к центру.

**Глубину НЕ трогаем (default 0).** Изначально планировался `setDepth(-1)`, но при
реализации выяснилось: пол арены (`tileSprite 'floor'`) и живые враги оба на **depth 0**
(упорядочены порядком вставки), поэтому `depth -1` спрятал бы труп **под полом**. Default
depth 0 (как у партиклов/damage-text) рендерит труп над полом корректно.

Состояние: `timer`, `done` (флаг для фильтра).

## Анимация «падение/распад вниз»

Длительность `C.DEATH_FX.DURATION` (~0.3с). `k = timer / DURATION`, клампится в [0,1]:

- **Вспышка → тьма:** при `k < FLASH_K` (~0.15) — белый тинт `0xFFFFFF`;
  далее лерп тинта от белого к тёмному `DARK_TINT` (`0x2D2D37` = rgb(45,45,55)).
  Лерп тинта делаем покомпонентно (раскладываем hex на r/g/b, интерполируем, собираем
  обратно) внутри `DeathFx`.
- **Оседание:** `scaleY` от слепка → `slepok * END_SCALE_Y` (~0.2); `scaleX` слегка
  раздувается (`* (1 + WIDEN * (1-k))`, `WIDEN` ~0.1), чтобы «расплющивалось».
- **Затухание:** alpha = 1 до `k = FADE_START` (~0.55), затем линейно в 0.
- На `k >= 1` → `done = true`.

`release()` прячет спрайт (`setVisible(false)`); пул переиспользует его, `reinit`
переустанавливает текстуру/слепок/тинт/alpha/scale/origin/позицию.

## Краевые случаи

- **Хитфлеш на момент смерти.** Враг может умереть с красным тинтом хитфлеша. Слепок
  стартует с собственной белой вспышки и ведёт свой тинт — прежний тинт игнорируется
  (труп — отдельный спрайт). Проблемы нет.
- **Пауза.** Апдейт трупов идёт в общем цикле, гейтится тем же состоянием, что и
  партиклы — на паузе замирают. Допустимо, как у партиклов.
- **Mosher split.** Осколки (`split`) порождаются на смерти мошера независимо; труп
  мошера всё равно порождается. Конфликта нет.
- **Сброс рана / смена сцены.** `_releaseAll(this.deathFx, 'deathFx')` чистит активные
  трупы по образцу `dmgTexts`/`particles`.

## Что НЕ делаем (YAGNI)

- Никаких отдельных анимаций для боссов (по решению — боссов не трогаем).
- Никаких новых ассетов/спрайт-листов: труп — это слепок текущей текстуры врага.
- Не меняем сбор лута, души, shake, партиклы — они уже работают.

## Проверка

Локально: `python -m http.server 8000` → открыть игру, начать забег.
Убедиться, что обычные враги (включая гоблина, мошера, сабвуфер) при смерти **оседают
вниз с потемнением и затуханием**, а боссы по-прежнему исчезают резко под взрыв + shake.
