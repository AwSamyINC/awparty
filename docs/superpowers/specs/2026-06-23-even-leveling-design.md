# Дизайн: ровная прокачка уровня (линейная XP-кривая)

Дата: 2026-06-23

## Проблема

Сейчас прокачка «размазана»: в начале главы уровень набирается очень быстро
(особенно «пачкой» на убийстве первого босса), а после ~12–13 уровня левелы
почти перестают приходить — теряется ощущение прогрессии.

Причина — **экспоненциальная цена уровня против почти плоского дохода XP**:

- Каждый гем = ровно **1 XP** ([src/scene.js](../../../src/scene.js) ~695,
  `p.gainXP(1)`); обычный враг роняет 1 гем, гоблин — 3, боссы — 15/17/20 разом.
- Крива́я опыта экспоненциальная: старт `xpToNextLevel = 5`, далее `*= 1.5`
  каждый уровень ([src/scene.js](../../../src/scene.js) ~939).
- Приток XP почти постоянный: спавн растёт лишь на ~15% за этап
  (`1.15^(stage-1)` в [src/spawner.js](../../../src/spawner.js) ~58),
  т.е. ~1.7 XP/с в начале забега и ~2.4 XP/с в конце.

Итог — цена уровня обгоняет доход геометрически:

| Уровень | XP на уровень (×1.5) | ≈ время при ~2 XP/с |
|---|---|---|
| 1→2 | 5 | ~3 с |
| 5→6 | 25 | ~13 с |
| 10→11 | 192 | ~1.5 мин |
| 13→14 | 649 | ~5 мин |
| 15→16 | 1460 | ~12 мин |

## Цель

Ровный темп левел-апов **на протяжении всего забега**, привязанный к боссам:
**lvl 5 к боссу 1, lvl 10 к боссу 2, lvl 15–20 к боссу 3.** Систему сбора опыта
**не меняем** — гемы так же падают/притягиваются/подбираются, «1 гем = 1 XP».
Меняем только **математику кривой стоимости**.

## Решение: линейная (арифметическая) кривая

Заменяем экспоненту на арифметический рост:

```
cost(L) = XP_BASE + XP_STEP · (L − 1)
```

Значения (подбираются плейтестом): **`XP_BASE = 20`, `XP_STEP = 1`**.

### Калибровка под боссов

Целевые вехи: **lvl 5 у босса 1, lvl 10 у босса 2, lvl 15–20 у босса 3.**
Боссы гейтятся числом убийств (`BOSS_KILL_REQ = [85, 95, 110]`), а убийство ≈
1 гем = 1 XP, поэтому накопленный XP к боссу предсказуем:

| Босс | ≈ XP доступно к боссу (килы + дроп + гоблины/мини) | Цель | cum(XP) для цели |
|---|---|---|---|
| Б1 | ~95 (85 килов + мелочь) | lvl 5 | 20+21+22+23 = **86** |
| Б2 | ~230 (+дроп Б1 15, +95 килов, +гоблины) | lvl 10 | **216** |
| Б3 | ~388 (+дроп Б2 17, +110 килов, +гоблины) | lvl 15 | **371** |

Накопленный XP до цели чуть меньше доступного → уровень берётся как раз перед
боссом. Б3 садится на нижний край (lvl 15); если реальный доход выше прикидки
(сабвуферы/мошеры/хайпмены тоже роняют гемы) — будет 16–18.

**Ранний «взрыв» на первом боссе укрощён.** Дроп босса (+15 гемов = +15 XP) при
цене уровня ~20 XP — это ~0.75 уровня, а не 2–3, как на старой экспоненте.
Небольшой бонусный «толчок» от босса остаётся (это уместная награда за килл), но
не ломает прогрессию. Поэтому дроп с боссов **трогать не нужно** (а ещё это
убережёт от правки монет, которые падают в том же цикле, см.
[src/scene_combat.js](../../../src/scene_combat.js) ~158–184).

## Изменения в коде (4 точки)

### 1. constants.js — две новые константы

В объект `C` (рядом с прочими игровыми константами) добавить:

```js
XP_BASE: 20,   // цена 1-го уровня (XP)
XP_STEP: 1,    // прирост цены за каждый следующий уровень (XP)
```

### 2. entities.js (~15) — инициализация игрока

```js
this.xpToNextLevel = C.XP_BASE;   // было: = 5
```

`C` уже глобальна и используется в этом файле
([src/entities.js](../../../src/entities.js) ~272) — доступ есть.

### 3. scene.js (~356) — сброс рана

```js
p.xpToNextLevel = C.XP_BASE;      // было: p.xpToNextLevel = 5;
```

(строка `p.level = 1; p.currentXP = 0; p.xpToNextLevel = 5; p.shootCooldown = 0.45;`)

### 4. scene.js (~939) — `triggerLevelUp()`

```js
p.xpToNextLevel += C.XP_STEP;     // было: p.xpToNextLevel *= 1.5;
```

Инкрементальный паттерн (`+= STEP`) точно ложится на формулу: после левел-апа в
`L` поле равно `XP_BASE + XP_STEP·(L−1) = cost(L)`. Заодно убираем магическую `5`,
размазанную по двум файлам. Всё остаётся **целочисленным** (раньше `×1.5` давал
дробные значения вроде 7.5).

## Что НЕ трогаем

- Сбор опыта: дроп гемов, притяжение магнитом, подбор (`gainXP(1)`),
  «1 гем = 1 XP».
- Дроп с боссов (15/17/20 гемов) — с линейной кривой темп уже не ломается.
- Карты левел-апа, способности.
- HUD-бар XP: `xpPct = currentXP / xpToNextLevel`
  ([src/hud.js](../../../src/hud.js) ~166) работает с любыми значениями.
- Хил `+20` HP за уровень в `triggerLevelUp` — не меняем.

## Риски и тюнинг

- Единственный риск — **числа**. Если 1-й уровень покажется медленным —
  снизить `XP_BASE`; если поздние уровни приходят слишком быстро/медленно —
  крутить `XP_STEP`. Обе константы в одном месте.
- Регрессий по логике практически негде взяться: правки точечные, тип данных
  становится «чище» (целые вместо дробных).

## Затрагиваемые файлы

- `src/constants.js` — `XP_BASE`, `XP_STEP`.
- `src/entities.js` — инициализация `xpToNextLevel` (~15).
- `src/scene.js` — сброс рана (~356) + `triggerLevelUp` (~939).
- `index.html` — `ASSET_VER` +1 (порядок модулей не трогаем).

## Дефолты (зафиксированы)

- Формула: **линейная** `cost(L) = XP_BASE + XP_STEP·(L−1)`.
- Числа: `XP_BASE = 20`, `XP_STEP = 1` — калибровка под боссов (lvl 5 / 10 / ~15),
  тюнятся плейтестом.
- Дроп с боссов и система сбора опыта — **без изменений**.
